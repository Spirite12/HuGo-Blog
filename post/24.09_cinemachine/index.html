前言

Cinemachine 是 Unity 提供的一个强大的摄像机控制插件，专门用于管理和优化摄像机在游戏或动画中的移动和跟踪。它能够极大地简化摄像机的管理与控制，使得开发人员可以更专注于游戏或动画的其他重要部分；

环境安装

  • 路径：菜单栏 -> Window -> Package Manager

  • 窗口：右上角搜索 Cinemachine -> 右下角点击安装 Install

  • 示例：安装成功后，可以在窗口内看到示例 Samples，可以点击安装查看官方示例；

精要介绍

一、移动路径

  • 设定路径
    • 基类
      • CinemachinePathBase：世界空间路径的抽象基类，适用于摄影机移动轨道；
    • 子类
      • CinemachinePath ：由一组点组成一个移动轨道路径，每个点都有位置、切线和滚动设置。点与点之间执行贝塞尔插值，以获得平滑连续的路径；
      • CinemachineSmoothPath ：具备 CinemachinePath 功能的同时保证一阶和二阶连续性，这意味着使得整个路径会更加平滑和连续性，推荐使用这个；
  • 执行路径
    • CinemachineDollyCart ：执行播放设定好的路径；

二、摄像机组件

  • 主相机
    • Cinemachine Brain ：虚拟相机的管理类。监控场景中所有活动的虚拟相机，并选择其中一个虚拟相机来控制 Unity 相机；
  • 虚拟相机
    • CinemachineVirtualCamera：基础虚拟相机。控制位置、旋转等属性设置跟随目标；
    • CinemachineFreeLook ：自由视角相机。组件管理三个水平环形轨道以围绕目标；
    • CinemachineStateDrivenCamera：状态驱动相机。控制一个 Animator 组件；将拥有的子虚拟相机都分配给动画状态；作用为切动画来达到切换虚拟相机的激活；
    • CinemachineClearShot : 分镜摄像机。在拥有的子虚拟相机内，以选取规则选择最佳质量的目标相机并加以激活，以确保能清晰地看到目标；
      • 选取规则：先选取距离靠近相机的，若有多台则为查找高优先级更高的相机；
    • CinemachineBlendListCamera：混合列表相机。按序播放子虚拟相机列表；适合剧情播放；
    • CinemachineMixingCamera：混合相机。使用子虚拟相机的加权平均值来计算最终相机的位置和其属性；

三、辅助组件

  • CinemachineTargetGroup：目标组。将多个 GameObjects 视为单个 Look At 目标，使得虚拟相机可以看向此单个目标；
  • CinemachineTriggerAction：触发器。监听目标进入和退出区域的函数调用，代码实现为碰撞盒的触发器函数OnTriggerXXX，条件为：需要设置 Collider 的 IsTrigger 为 True；

工程说明

以下二级菜单为打开的场景名

Trigger volume

  • 可接触知识点

    • 移动路径：CinemachinePath、CinemachineDollyCart；
    • 摄像机组件：CinemachineBrain、CinemachineVirtualCamera；
    • 辅助组件：CinemachineTargetGroup、CinemachineTriggerAction；

  • 场景说明

    1. CinamachineBrain ：主虚拟相机

      • 路径：MainCamera
      • 说明：监控场景内所有的虚拟相机，一般与主 Camera 组件挂载在一起

    2. CinemachineTriggerAction ：触发器

      • 路径：Environment -> DollyTrackTrigger

      • 说明：当进入当前物体的触发器区域的时候，会响应 OnObjectEnter，而离开当前区域的时候，会响应 OnObjectExit；需要2个当前脚本的原因是需要控制到2个物体；

        • Action

          • Activate 和 Deactivate 是 Target 物体的显隐；
          • Enable 和 Disable 是 Target 组件上的脚本显隐；

      • 代码：脚本继承 Monobehaviour，从脚本内可查看到 OnObjectEnter 和 OnObjectExit 的实现方式是由碰撞盒或触发器的 Enter 和 Exit 响应的；

    3. Cinemachine Dolly Cart ：路径执行

      • 缘由：点击 Cinemachine Trigger Action 脚本上的 OnObjectEnter 上的 Target 实例 DollyCart 引用，并点击场景内的物体，可以发现 CineMachine Dolly Cart 的组件；
      • 路径：Environment -> DollyTrackWithCart -> DollyCart
      • 说明：根据 Path 提供的一系列位置路径，按更新方式赋值当前物体的位置；
      • 参数：
        • Path：指定路径（在下一点内说明）；
        • Update Method ：更新方式
          • Update ：每帧更新；
          • Fixed Update ：固定时间更新；
          • Late Update ：Update后更新，适合摄像机位置推进；
        • Position Units：位置计量单位
          • Path Units：使用路径上的航点，0为第一个航点，1为第二个，以此类推；
          • Distance：使用路径全距离处理；
          • Normalized：归一化处理，0为开始，1为结束；
        • Speed ：移动速度
        • Position ：初始在哪个位置上；

    4. CinemachinePath：绘制路径

      • 缘由：点击 Cinemachine Dolly Cart 脚本上的 Path DollyTrack 实例；便可查看到 CinemachinePath 脚本；
      • 路径：Environment -> DollyTrackWithCart -> DollyTrack
      • 说明：绘制一系列的移动路径轨道；
      • 参数：链接

    5. CinemachineVirtualCamera：虚拟相机

      • 说明：继承 CinemachineVirtualCameraBase，虚拟相机内的基础相机；
      • 缘由：点击 Cinemachine Trigger Action 脚本上的 OnObjectEnter 上的 Target 实例 CM vcam DollyTrack 引用，并点击场景内的物体，可以发现 CinemachineVirtualCamera 的组件；
      • 路径以及说明：
        • CM vcam DollyTrack：跟随DollyCart的虚拟相机，当进入中心蓝色触发区域后，会激活并使用当前虚拟相机；
        • CM vcam Overview：跟随玩家的虚拟相机；当离开中心蓝色触发区域后，会使用当前虚拟相机；
      • 参数：
        • Priority：优先级
        • Follow ：虚拟相机跟随的物体；
        • LookAt：虚拟相机朝向的物体；
        • Body：更改位置 链接
          • Transposer：可以点击 Transposer 场景查看其效果， 链接
        • Anim：旋转相机 链接
        • 其他：链接

    6. CinemachineTargetGroup：虚拟相机目标组

      • 缘由：点击 CM vcam DollyTrack -> CinemachineVirtualCamera -> Look At -> TargetGroup1 引用，并点击场景内的物体，可以发现 CinemachineTargetGroup 的组件；
      • 说明：由于虚拟摄像机的 Look At 只能看向一个目标，为了能同时看向多个目标，则需要当前组件提供支持；
      • 参数：
        • Target：可以添加多个目标；
          • Weight：计算当前物体的权重，不能为负数；
          • Radius：计算当前物体的半径，不能为负数；
        • 其他：链接
