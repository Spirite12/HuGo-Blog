前言

从20年就开始写UI框架代码了，但是每次都是写几个月就中断了。

今年上半年狠心就给自己定了个S级的大目标，至少写个 0.1版本的框架代码并上传！

项目地址

  • https://gitlab.com/Spirite12/dcFrame

框架目录

  • Editor
    • ADF说明：因为根目录有 Adf.Unity.Frame，因此额外开个ADF，防止无法打包；
    • UIManager：UI框架相关的Editor代码（略过不介绍）
  • Singleton：单例基类集合点
    • 说明：目前有 Mono、Class 类单例基类；
  • Utility：工具类集合点
    • 说明：目前有 StringUtil、 TimeUtil 供当前框架使用，后续有不断完善；
  • UIManager：UI框架脚本

    • UI主体脚本：

      • MainUI：主UI脚本，挂载在主场景，脚本会在运行游戏就执行该脚本；
        • 提供GC回收、低内存事件通知、切换后台事件通知；
      • UIMgr：UI管理脚本，堆栈分配、层级设置、界面的显示隐藏处理；
      • UIBase：界面的基类
        • 提供：打开、关闭、加载、释放、层级设置等函数支持；
      • UIBaseSingleton：界面基类的单例脚本，所有界面子类需继承该脚本；
      • IUIItem：子预制体的基类，一般不用手动创建，可通过 Ctrl Script 工具创建脚本；

    • 辅助脚本：

      • UIAdaptive：安卓、IOS 适配脚本；
      • UIConst：UI常量脚本；
      • UIOrder：界面层级脚本，会获取当前界面层级值，并赋值给子节点下所需赋值的物体；
      • UITool：工具类，目前提供了界面队列打开的函数支持；

    • 预制件编辑器脚本：

      • UIMono：预制体根节点脚本，一般是所需要的引用的变量脚本，也提供了相关定位工具；一般不用手动创建，可通过 Ctrl Script 工具创建脚本；

      • UIAutoRef：预制件自动引用模板脚本，作用是在预制件上显示Icon来提示该物体代码有引用到；

      • UICreateItem：子预制件脚本，需要挂载在父预制体需要实例化生成子预制件的物体上；

      • UILockData：预制件上锁脚本。上锁预制件会留下机名，作用是：防止冲突，提示其他人此预制体有人在修改；

        • 注意：目前请求和获取上锁数据均未写，这两处应由项目内内网与服务端接入，因此不同地方的地址是不同的；

使用说明

  1. 创建预制体所需脚本：

    • 首先先准备一个预制件，在 Project 面板右击该预制件，选择 Create -> Ctrl Script，并点击；

      • 若选择的不是预制件，则 Ctrl Script 选项则置灰不让点击；

    • 出现弹窗：

      • 自动填充的选项：以下不建议更改参数
        1. Select Prefab ：当前所选择的预制体；
        2. Mono、Ctrl FilePath：Mono 脚本 与 Ctrl 脚本的路径：
        3. Create FileName：所创建的脚本名
      • 可选项：
        1. IsPanel：是否是父预制体
          • 是：UIBase Type：设置弹窗类型
          • 否：IsAddListenerFunc：是否自动帮忙添加监听事件空函数；
        2. IsFocusCtrl：创建Ctrl脚本后，是否选中Ctrl脚本根节点；
      • 工具截图：

    • 点击Create Script 后，该工具会执行以下操作：

      1. 在对应文件夹下会创建 Mono 脚本（继承UIMono），并挂载在当前预制体上；

      2. 在对应文件夹下会创建 Ctrl 脚本（继承UIBaseSingleton），并根据模板会生成一套基础UIBase 所需代码；

      3. 往该预制体上，添加 UIOrder 脚本，使其预制件有层级显示，UIOrder 脚本会绑定添加 Canvas 和 Graphic Raycaster 脚本；

  2. 自动引用、删除引用代码中所需操作的物体：

    • 继承至 UIMono 的脚本会显示精灵球；

    • 查看 UIAutoRef 所填写的参数模板，若在预制体中若前缀匹配 Prefix ，则在预制体中显示小蓝标；

    • 根据步骤1，在预制件根节点有个 UIMono脚本，该脚本显示如下：

    • 若点击自动引用，则工具会把小蓝标的物体全部添加到该脚本里

      • 注意：该工具会自动屏蔽挂了 子预制体 UICreateItem 的脚本物体以及该子物体；
      • 重点：工具会找到 当前对应的 Ctrl 脚本，查找当前引用有 btn（按钮）开头的，则往 Ctrl 脚本中添加相关监听函数；（同理 也会帮忙挂载了UICreateItem 脚本，新增相对应变量，和销毁函数）
      • 方便：减少开发人员手动写变量、挂载实例引用的操作、手动写按钮监听函数、子预制件销毁函数；

    • 若点击删除引用，则该脚本则会删除小蓝标所标识物体名的变量，同时也会删除 Ctrl 脚本中 点击引用 所帮忙创建的相对应的脚本；

  3. 创建子预制体所需脚本：

    • 为了减少开发人员手动写子预制件创建代码和维护代码，才有了这个工具；

    • 在所需帮忙创建子预制体的物体上挂载 UICreateItem 脚本，并设置所需实例化的预制件：

      • 在游戏运行、将预制体拖到 Hierarchy 面板上，代码都会自动帮忙创建子预制件；

    • 因为 UICreateItem 继承至 UIMono，因此也可以自动引用和删除引用；

      • 与之不同的是：子预制件的基类是：IUIItem，自动引用会在相对应位置添加按钮监听和销毁代码

  4. UIMgr 所帮忙操作的事情：

    • 抽象介绍：

      • 全屏界面：类似是系统的主界面；
      • 非全屏界面：类似是覆盖在当前主界面上的弹窗；
      • 首先打开的主预制体则是全屏界面，只要接下来打开的界面是非全屏类型，则在该全屏界面上不断叠加并显示。当打开一个新全屏界面后，则关闭当前所有界面，再以全屏的界面的操作形式继续下去；当关闭一个全屏界面后，则会关闭当前全屏上打开的所有非全屏界面，并打开之前关闭的界面直至到全屏界面；

    • 预制件的层级管理，而预制体又会以当前层级管理自己子预制体；

    • 预制件的堆栈管理，在打开新的全屏界面，则隐藏之前的所有界面。当关闭当前界面后，又会自动打开之前的所展示的界面，无需手动写代码；

      • 因此预制体提供了 OnHide（监听移除函数）、OnRelease（数据销毁函数），需拆分清楚则可实现以上机制；

  5. 其他的小功能就不介绍了；
