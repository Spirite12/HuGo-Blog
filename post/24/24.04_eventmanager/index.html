前言

  • 在游戏的实际开发内会包含许多的事件方法，比如：1：24/5点红点刷新；2：战斗胜利/失败触发；3：玩家数值变更；等等；
  • 在游戏开发过程中，由于事件是由类声明的，因此在开发过程中倘若其他类也需要监听，则需要知道当前事件是在哪个类内声明才可以注册并发放事件；而当工程量逐步扩大的情况下，事件散落在各个脚本，显示会比较杂乱无章，因此希望有个统一管理的事件脚本；

一、事件常量类

  1. 事件常量类作用：统一需要声明的事件名称在一个脚本内，方便后续多个脚本需要共同一个事件的时候，无需在各个脚本内查找，代码如下：

  2. nameof ：让变量、类型或成员的名称作为字符串常量；

  3. Q1：正常只需要一个 EventConst 脚本便可实现事件常量类，此处使用到了2个脚本的原因是：

    A：EventConstBase 脚本处在框架的工程内，为了不让项目实际事件都写在框架脚本内，目的让框架工程和项目工程做到解耦，因此需要拆分2个脚本，一个供框架（EventConstBase）使用，一个供项目（EventConst）使用；并且 EventConst 作为 EventConstBase 的子类，使得项目内脚本也可以拿到框架事件；

  4. Q2：C# 提供 partial 关键词达到一个类可以拆分成多个脚本来写，此处不用这个关键词的原因是：

    A：当前框架工程和项目工程是处在不同的dll文件内，这2个地方已经用 Assembly Definition Reference 达到隔离；而partial 的使用说明中有一处如下说到，因此不能使用此关键词；

    要成为同一类型的各个部分的所有分部类型定义都必须在同一程序集和同一模块（.exe 或.dll 文件）中进行定义。分部定义不能跨越多个模块；

    using System;
using System.Collections.Generic;

namespace Module {
    public class EventManager {

        /// <summary>
        /// 监听不需要参数传递的事件
        /// </summary>
        public static void AddListener(string eventName, Action callback) {
            AddListenerBase(eventName, callback);
        }
        /// <summary>
        /// 添加事件监听（1个参数）
        /// </summary>
        public static void AddListener<T>(string eventName, Action<T> callback) {
            AddListenerBase(eventName, callback);
        }
        /// <summary>
        /// 添加事件监听（2个参数）
        /// </summary>
        public static void AddListener<T1, T2>(string eventName, Action<T1, T2> callback) {
            AddListenerBase(eventName, callback);
        }
        /// <summary>
        /// 添加事件监听（3个参数）
        /// </summary>
        public static void AddListener<T1, T2, T3>(string eventName, Action<T1, T2, T3> callback) {
            AddListenerBase(eventName, callback);
        }
        /// <summary>
        /// 添加事件监听（4个参数）
        /// </summary>
        public static void AddListener<T1, T2, T3, T4>(string eventName, Action<T1, T2, T3, T4> callback) {
            AddListenerBase(eventName, callback);
        }

        /// <summary>
        /// 移除不需要参数的事件
        /// </summary>
        public static void RemoveListener(string eventName, Action callback) {
            RemoveListenerBase(eventName, callback);
        }
        /// <summary>
        /// 移除对应的事件监听（1个参数）
        /// </summary>
        public static void RemoveListener<T>(string eventName, Action<T> callback) {
            RemoveListenerBase(eventName, callback);
        }
        /// <summary>
        /// 移除对应的事件监听（2个参数）
        /// </summary>
        public static void RemoveListener<T1, T2>(string eventName, Action<T1, T2> callback) {
            RemoveListenerBase(eventName, callback);
        }
        /// <summary>
        /// 移除对应的事件监听（3个参数）
        /// </summary>
        public static void RemoveListener<T1, T2, T3>(string eventName, Action<T1, T2, T3> callback) {
            RemoveListenerBase(eventName, callback);
        }
        /// <summary>
        /// 移除对应的事件监听（4个参数）
        /// </summary>
        public static void RemoveListener<T1, T2, T3, T4>(string eventName, Action<T1, T2, T3, T4> callback) {
            RemoveListenerBase(eventName, callback);
        }

        /// <summary>
        /// 事件触发（不需要参数的）
        /// </summary>
        public static void DispatchEvent(string eventName) {
            if (eventDic.ContainsKey(eventName)) {
                foreach (Delegate callback in eventDic[eventName]) {
                    (callback as Action)?.Invoke();
                }
            }
        }
        /// <summary>
        /// 事件触发（1个参数）
        /// </summary>
        public static void DispatchEvent<T>(string eventName, T info) {
            if (eventDic.ContainsKey(eventName)) {
                foreach (Delegate callback in eventDic[eventName]) {
                    (callback as Action<T>)?.Invoke(info);
                }
            }
        }
        /// <summary>
        /// 事件触发（2个参数）
        /// </summary>
        public static void DispatchEvent<T1, T2>(string eventName, T1 info1, T2 info2) {
            if (eventDic.ContainsKey(eventName)) {
                foreach (Delegate callback in eventDic[eventName]) {
                    (callback as Action<T1, T2>)?.Invoke(info1, info2);
                }
            }
        }
        /// <summary>
        /// 事件触发（3个参数）
        /// </summary>
        public static void DispatchEvent<T1, T2, T3>(string eventName, T1 info1, T2 info2, T3 info3) {
            if (eventDic.ContainsKey(eventName)) {
                foreach (Delegate callback in eventDic[eventName]) {
                    (callback as Action<T1, T2, T3>)?.Invoke(info1, info2, info3);
                }
            }
        }
        /// <summary>
        /// 事件触发（4个参数）
        /// </summary>
        public static void DispatchEvent<T1, T2, T3, T4>(string eventName, T1 info1, T2 info2, T3 info3, T4 info4) {
            if (eventDic.ContainsKey(eventName)) {
                foreach (Delegate callback in eventDic[eventName]) {
                    (callback as Action<T1, T2, T3, T4>)?.Invoke(info1, info2, info3, info4);
                }
            }
        }

        /// <summary>
        /// 添加事件
        /// </summary>
        private static void AddListenerBase(string eventName, Delegate callBack) {
            if (eventDic.ContainsKey(eventName)) {
                eventDic[eventName].Add(callBack);
            }else {
                eventDic.Add(eventName, new List<Delegate>(){ callBack });
            }
        }

        /// <summary>
        /// 移除事件
        /// </summary>
        private static void RemoveListenerBase(string eventName, Delegate callBack) {
            if (eventDic.TryGetValue(eventName, out List<Delegate> eventList)) {
                eventList.Remove(callBack);
                if (eventList.Count == 0) {
                    eventDic.Remove(eventName);
                }
            }
        }

        /// <summary>
        /// 清空事件
        /// </summary>
        public static void Clear() {
            eventDic.Clear();
        }

        private static readonly Dictionary<string, List<Delegate>> eventDic = new Dictionary<string, List<Delegate>>();
    }
}

  2. T：利用泛型来达到不同的数据类型传参；

  3. 目前提供能够匹配最多 4 个参数的无返回值方法，倘若需要更多，则需要对当前代码进行补充；

  4. Q：是否可以用枚举作为 Key 来达到字典的存储（需要在 EventConst 更改为枚举做法）：

    A：可以做，但不建议的理由如下：

    • 由于框架和项目是分开的，因此枚举没办法在2个脚本内定义，倘若只有1个脚本的是可以的；
    • 枚举作为字典 Key 的时候会在查询时产生装箱的问题，在性能上可能会有一点点损耗；

    using System.Collections.Generic;
using Module;
using UnityEngine;

namespace Base {
	public class MainUI : MonoSingleton<MainUI> {
		private void Awake() {
			EventManager.AddListener(EventConst.FiveTime, EventMethods);
            EventManager.AddListener<int>(EventConst.FiveTime, EventMethods1);
            EventManager.AddListener<string, List<int>>(EventConst.FiveTime, EventMethods2);
        }

        private void OnDestroy() {
			EventManager.RemoveListener(EventConst.FiveTime, EventMethods);
            EventManager.RemoveListener<int>(EventConst.FiveTime, EventMethods1);
            EventManager.RemoveListener<string, List<int>>(EventConst.FiveTime, EventMethods2);
            EventManager.Clear();
		}

        private void Update() {
            if (Input.GetKeyDown(KeyCode.A)) {
                EventManager.DispatchEvent(EventConst.FiveTime);
            }
            if (Input.GetKeyDown(KeyCode.S)) {
                EventManager.DispatchEvent(EventConst.FiveTime, 5);
            }
            if (Input.GetKeyDown(KeyCode.D)) {
                EventManager.DispatchEvent(EventConst.FiveTime, "文本1", new List<int>(){1, 2, 3});
            }
        }

        public void EventMethods() {
			Debug.LogWarning("收到事件发放");
        }

        public void EventMethods1(int number) {
            Debug.LogWarning("收到事件发放,数值为" + number);
        }

        public void EventMethods2(string str1, List<int> intList) {
            int value = 0;
            foreach (var intValue in intList) {
                value += intValue;
            }
            Debug.LogWarning("收到事件发放, 文本为：" + str1 + "\n总和为：" + value);
        }
    }
}

  4. 由控制台输出结果可知：

    1. 当触发同一个事件的时候，当前事件绑定了多重不同的参数，而发放事件的时候，只有相同参数的所匹配的方法才能被响应；
    2. 在写 EventManager.AddListener 的时候需要在销毁的时候写 EventManager.RemoveListener来达到移除事件，否则会出现切换场景或者实例销毁了，但是事件还存在的情况，导致内存泄漏；
