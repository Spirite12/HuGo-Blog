前言

以下来自官方的说明：

精灵图集 (Sprite Atlas) 是一种将多个纹理合并为一个组合纹理的资源。Unity 可以调用此单个纹理来发出单个绘制调用而不是发出多个绘制调用，能够以较小的性能开销一次性访问压缩的纹理；

本文 Unity 版本：2021.3.6f1c1

使用 SpriteAtlas 的好处有：

  • 减少 Draw Call ：通过将多张图片合并到一个大的 atlas 中，渲染时只需一个 draw call，这可以显著提升性能；
  • 减少 GPU 负担 ：将多张不同规格的图片，使用图集打包成一个2的幂次方的图集，可减少 GPU 处理负担，方便渲染合批；
  • 减少包体大小 ：将多张不同规格的图片，使用图集打包成一个2的幂次方的图集，有利于Unity压缩图片，更能节省空间；

导入图集

安装 2D Sprite

  • 点击菜单栏 Window -> Package Manager ；

  • 在窗口内查找 2D Sprite 或者 2D （这里面包含了 2D Sprite），并点击 Install 进行安装；

开启 Sprite Packer

  • 点击菜单栏 Edit -> Project Settings；

  • 点击左侧栏名为 Editor 的栏目，并再右侧找到 Sprite Packer ，在 Mode 不要选择 Disabled 便可；

    • Mode 选择说明请点击 22.3文档 、21.3文档

新建图集

  • 右击 Project 面板，选择 Create -> 2D -> Sprite Atlas
  • 此时 Unity 会新建一个名为 New Sprite Atlas 的图集资源；

参数说明

  • 文档说明：链接

  • Type：默认 Master ：主图集；

    • 可选 Variant ：变体图集，需要指定主图集；作用是：针对不同的硬件设备（高、中、低等性能设备），可以对其进行 Scale 属性缩放分辨率显示已达到性能适配需求；

  • Include in Build ：勾选此框可将 Sprite Atlas Asset 包含在当前构建中。此选项默认启用；

  • Allow Rotation ：勾选则允许 Sprite 在图集中进行旋转；

    • 勾选的情况：可最大化利用图集的空间；
    • 不选的情况：如果图集内存在精灵贴图在 UI 面板内显示，则无需勾选此项，因为图集内的旋转会同样导致在 UI 面板上渲染的时候进行旋转；

  • Tight Packing ：勾选则根据 Sprite 轮廓（而非默认的矩形轮廓）打包 Sprite；

    • 勾选的情况：可最大化利用图集的空间；
    • 不选的情况：由于勾选会根据 Sprite 轮廓进行图集打包，而 Sprite 贴图在 UI 面板渲染的是一张矩形贴图，因此会导致在 UI 面板的矩形区域内会渲染到图集内挨近的贴图；

  • Alpha Dilation ：勾选会提高 Sprite 的渲染质量，例如：在 Sprite 放大时防止出现锯齿边缘；

  • Padding ：每个 Sprite 贴图之间的像素值，防止贴图重贴；默认为4像素；

  • Read/Write ：是否开启可访问纹理数据；若开启次属性，Unity 将创建纹理数据的副本。这会使纹理资源所需的内存量翻倍，并会对性能产生负面影响；

  • Generate Mip Maps ：勾选此框可启用mipmap生成；

    • 开启 MipMaps ： 会包含4张不同精度的纹理贴图；在游戏场景内，当摄像头离当前贴图越近，则会使用精度越高的贴图。反之亦然；弊处是：开启会在磁盘和内存中的大小增加 33% ；

  • sRGB ：如果勾选该选项，则使用 sRGB 格式；否则，将使用线性格式，默认勾选；

  • Filter Mode ：选择纹理在被 3D 变换拉伸时如何进行纹理过滤模式。默认选项为 Bilinear；

    • Point ：从近处看，纹理显得块状；
    • Bilinear ：近看纹理显得模糊；
    • Trilinear ：与 Bilinear 类似，但纹理在不同的 MIP 级别之间也会变得模糊。

  • Show Platform Settings For：可以设置各个平台支持的参数

    • MaxTextureSize ：设置最大导入纹理尺寸像素，此参数根据实际项目需求进行设置；

      • 若设置过大：则会造成图集的内存浪费；
      • 若设置像素小于实际图集内的像素，图集则会拆分多个子图集；即是：假设当前图集内可以存放16张贴图；当出现有17张贴图的时候，图集则会划分为2页，第一页存放16张，第二页存放1张贴图；以此类推；

    • Format ：Automatic是默认选项，此选项会自动选择最适合的像素格式。其他可用格式列表取决于平台和纹理类型。平台特定覆盖的纹理格式参考。

    • Compression ：可以选择不同质量的压缩方式；None即为不压缩；不同的压缩选择会带来图集内的 Sprite 显示精度，也会带来图集的内存不一致；

      • 内存占用从大到小为：None -> High Quality -> Normal Quality -> Low Quality；
      • 精度显示从精到糊为：None -> High Quality -> Normal Quality -> Low Quality；

    • Use Crunch Compress ：勾选 Crunch 压缩可帮助纹理在磁盘上和下载时使用尽可能少的空间。Crunch 纹理可能需要很长时间才能压缩，但运行时解压缩速度非常快。

  • Objects for Packing ：可放入Texture2D、Sprite、文件夹。放入文件夹时会将文件夹下的所有Texture2D、Sprite放入图集；（推荐拖入文件夹，因为文件夹内新增可以直接更新，而拖入具体资源只能更新当前拖入的资源）

  • Pack Preview 按钮 ：会将 Objects for Packing 内存放的所有的纹理贴图都打包到一张图集内；

Draw Call

  1. 打开一个新场景，并运行游戏，并记入此时 Statistics 面板的数据；

    • Batches 默认为 2 ，是因为一个是天空盒一个是默认背景；

  2. 在 Project 栏目内新建 SpriteAtlas 文件夹，从网络上下载一些贴图素材并导入到当前文件夹内；素材链接

  3. 把这6张素材在 UI 场景内，并运行游戏，查看此时 Statistics 面板的数据；

    • 发现 Batches 由原来的 2 上涨到了 8 ，原因是添加了这 6 张贴图；

  4. SpriteAtlas 文件夹下新建 Sprite Atlas 图集，并对此设置基础参数；使得当前 6 张贴图打成一张图集；

  5. 重新运行游戏，查看 Statistics 面板数据；

    • 发现 Batches 由原来的 8 下降到了 3 ，原因是 6 张贴图进行了打图集的操作；因此 3 = 2 （空场景） + 1（图集）

动态图集

  1. 动态图集演示：链接
  2. 原理分析：
    • 动态图集的处理方式为先申请一张图集贴图，以这张图集去处理新增的纹理贴图、以及不定期的从图集内移除不使用的纹理贴图；当当前的图集不足以容纳新贴图的时候，则需新生成一张图集去处理新纹理贴图；以动态的方式去最大化的减少多张图集的使用；
    • 动态图集的核心算法为新纹理贴图当被需要生成的时候，此纹理贴图应该放在当前图集的哪个区域内，使得当前图集的使用率最大化；也便是模拟 Unity 上图集生成的规则；
  3. 功能说明：
    1. 动态图集的核心算法如下说明：链接
    2. Github项目中使用此算法的是：链接
      • 对此项目下载并尝试，可以得出同一图集内的 DrawCall 是会有变少；
  4. 优劣势：
    • 优势：
      • 动态图集自动生成，减少人为生成和管理图集的时间成本；
      • 动态图集按需加载，减少不确定通用贴图在多个图集内被加载导致所需内存上升；
    • 劣势：
      • 动态图集的生成和管理需要一定的代码计算量；可能会导致游戏卡顿；
      • 若项目内有提前打包的图集和动态图集一起使用，则需要合理划分规则；

工具

图集更新

  • 需求：当贴图内的 Objects for packing 指定的是文件夹，此时当这个文件夹内的贴图进行中变化的时候，需要开发者手动点击 Pack Preview 按钮，才可以更新图集内的贴图显示；对此当项目内图集的量开始多了起来的时候，将会是比较费事费力的行为；

  • 解决：需要一个按钮来达到所有图集的更新工具；

  • 代码：新建一个 Editor 文件夹，并在此文件夹内新建名为 SpriteAtlasEditor.cs 的脚本，添加如下代码；

    using System.IO;
using UnityEditor;
using UnityEngine;
using UnityEditor.U2D;

public class SpriteAtlasEditor : Editor {
    [MenuItem("Tools/SpriteAtlas/生成图集")]
    private static void PackAllAtlases() {
        // 清除 Sprite Atlas 缓存
        string pathAtlasCache = Path.Combine(Application.dataPath, "../Library/AtlasCache");
        if (Directory.Exists(pathAtlasCache)) {
            FileUtil.DeleteFileOrDirectory(pathAtlasCache);
        }
        // 重新打包图集
        SpriteAtlasUtility.PackAllAtlases(BuildTarget.Android, true);
        Debug.Log("图集生成成功");
    }
}

图集生成

  • 需求：当 UI 人员新增贴图不在图集范围内的时候，需要 UI 人员手动创建新图集，并且对其设置参数；然而在项目人员变更和手动操作的过程中，难免会出现参数设置错误；

  • 解决：增加对其需要新增图集的文件夹（因为单个贴图的图集作用不大，因此不实现这个功能）右键一键生成操作：生成操作会进行：1）生成图集到指定文件夹；2）对其进行一些基础的操作；

    using System.IO;
using UnityEditor;
using UnityEditor.U2D;
using UnityEngine;
using UnityEngine.U2D;

public class SpriteAtlasEditor : Editor {
    [MenuItem("Assets/Tools/Create SpriteAtlas", false)]
    public static void CreateCtrlWin() {
        var theSelection = Selection.activeObject;
        string path = AssetDatabase.GetAssetPath(theSelection);
        // 判断所选的文件夹下没有贴图
        string[] guids = AssetDatabase.FindAssets("t:Texture2D", new[] { path });
        if (guids.Length <= 0) {
            EditorUtility.DisplayDialog("Error info Win", $"当前所选文件夹下暂无纹理贴图", "关闭");
            return;
        }
        string pathTarget = path + "/" + Path.GetFileName(path) + ".spriteatlas";
        // 判断目标 Sprite Atlas 是否存在
        if (File.Exists(pathTarget)) {
            EditorUtility.DisplayDialog("Error info Win", $"当前目标路径已经存在 SpriteAtlas\n\n路径为：{pathTarget}", "关闭");
            return;
        }
        SpriteAtlas spriteAtlas = new SpriteAtlas();
        spriteAtlas.Add(new[] { theSelection });

        SpriteAtlasPackingSettings packingSettings = spriteAtlas.GetPackingSettings();
        packingSettings.enableRotation = false;
        packingSettings.enableTightPacking = false;
        packingSettings.enableAlphaDilation = true;
        packingSettings.padding = 4;
        spriteAtlas.SetPackingSettings(packingSettings);

        SpriteAtlasTextureSettings textureSettings = spriteAtlas.GetTextureSettings();
        textureSettings.readable = false;
        textureSettings.generateMipMaps = false;
        textureSettings.sRGB = true;
        textureSettings.filterMode = FilterMode.Bilinear;
        spriteAtlas.SetTextureSettings(textureSettings);
        // 针对平台设置参数
        TextureImporterPlatformSettings atlasSetting = spriteAtlas.GetPlatformSettings("DefaultTexturePlatform");
        atlasSetting.maxTextureSize = 1024;
        atlasSetting.textureCompression = TextureImporterCompression.CompressedHQ;
        atlasSetting.format = TextureImporterFormat.Automatic;
        spriteAtlas.SetPlatformSettings(atlasSetting);

        AssetDatabase.CreateAsset(spriteAtlas, pathTarget);
        AssetDatabase.Refresh();
    }

    [MenuItem("Assets/Tools/Create SptAtlas", true)]
    public static bool CreateCtrlWinValidate() {
        var theSelection = Selection.activeObject;
        string path = AssetDatabase.GetAssetPath(theSelection);
        return AssetDatabase.IsValidFolder(path);
    }
}

项目可行性

  1. 文件夹划分规则：
    • Common ：通用的、需要穿梭在各个模块内的纹理贴图存储；
    • Icon ：需要程序代码加载的一系列的纹理贴图为单位存储；
    • Language ：以各个语言为单位存储；
    • Module ：以各个功能模块为单位存储；
  2. 参数设置：
    • Max Texture Size ：
      • 该参数需要人工按照实际整体纹理贴图大小情况维护；
      • 参考值：该参数值默认为 1024，若遇到图整体大或小，则可相应更改为 2048 与 512；
      • 参考划分：
        • 对于 Common 文件夹：需要人工按照实际情况维护；
        • 对于 Icon 文件夹：建议采取 1024 。后续无需更改；
        • 对于 Language 文件夹：建议采取 1024 。后续无需更改；
        • 对于 Module 文件夹：建议采取 1024 。由于模块功能会存在多次迭代，每次迭代都会产生不同大小的纹理贴图；出于人工操作会忘记、且存在多次迭代，因此固定1024便可；
  3. 资源管辖：因背景图像素过大，因此背景图无需打图集。在这些文件夹的范围内不能存在背景图；
  4. 动态图集：
    • 如果项目内采用动态图集：
      • 则推荐取消 Common 文件夹的图集处理，因为通用的资源其实并没有有效的办法处理在同一图集内出现。因此当 Common 图集内资源过多，则会出现图集的页数也会变多，而加载到同一页数内的图集概率则并不高，此时图集所占用的内存会变高，因此推荐使用动态图集；
      • 按实际需求对 Language 文件夹作出处理，若多语言的资源在较少的地方出现且在同一时刻界面上出现的张数并不多的情况下，则推荐做动态图集处理；
