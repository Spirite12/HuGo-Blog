前言

由于 Lua 是弱类型语言，在某个数据内所持有数据内容并不能像强类型语言（例：C#）在Idea中明示出来。导致在开发过程中，多成员在使用同一份数据的时候会出现数据混淆、无法感知有哪些变量等等的问题。鉴于此情况，EmmyLua便很好的解决了这个情况；

注释

为了让 Idea 能识别出 EmmyLua 的注释说明，在函数、变量上方增加固定格式是：— @xxxxxxx，类似C#语言的注释说明一样，以下为参数注解：

  • — xxxx ：注释说明
  • — @class xxxxx ：说明这是一个类（可以实现继承）
  • — @type xxxx yyyy ：用来标注该类的变量（x为类型，y为说明）
  • — @field xxxx yyyy zzzz：用来标注该类的变量（x为变量名，y为变量类型，z为说明）（与 type 同用法，区别是该变量即使不在代码内使用）
    • — @field xxxx yyyy[] zzzz ：用来标注这是一个数组
    • — @field xxxx table<k, v> zzzz ：用来标注这是一个KV数组
  • — @public、— @protect、 — @private : 声明该函数类的共私有性质；
  • — @return xxxx ：返回的注释说明（x为返回数据类型）

--- 玩家类
--- @type Player 玩家类数据
local Player = Class()

--- @type number 是否移动
Player.isMove = false

--- @class Player
--- @field Name string 名称
--- @field Level number 等级
--- @field PackList Player.PackData[] 背包数据

--- @class Player.PackData
--- @field ID number ID
--- @field Num number 数量

--- 获取名称
--- @public
--- @return string
function Player:getName()
    return self.Name
end

--- 获取背包数据
--- @return number 数量
function Player:getPackNumById(packId)
    if not self.PackList[packId] then
        return 0
    end
    return self.PackList[packId].Num
end

return Player

后缀模板

EmmyLua插件中提供了后缀补充代码（Postfix Completion）的功能，该功能预设了一些模板，开发者可通过代码加"点"出模板内容，作出相对应的代码补充；

  • par ：对当前光标处增加中括号操作
  • var ：对当前光标处增加 Local 变量赋值操作
  • return ：对当前光标处增加 return 语句
  • if、if_not、if_nil、if_not_nil ：对当前光标处增加 if 语句的判断
  • for、for_i、for_p ：对当前光标处增加 for 语句
  • decrease、increase：对当前光标处变为 自减、自增算法语句

实战用法

  • 在开发过程中，每个预制体会有个脚本保存预制变量的引用（例如：C#脚本中用Mono挂在预制体保存）。而这些变量的申明因为在Lua中无法直观显示，则可额外生成一个文件，文件内注释一个Class和若干个预制体所用到的 Field 字段。这样在脚本里则可以引用到；
  • 用以上的思路，也可以应用在各个系统的 Proto文件、DataCtrl文件等等所需要引用的地方；
