前言

  • 项目工程内以 develop 分支为开发分支；线上分支则是从 develop 切出一条分支；
  • 当修复一端分支Bug的时候，则分别需要提交至两个分支内。此操作流程为：
    • 从一端切出新分支 -> 推送修复节点到当前分支 -> 推送当前分支到远端；
    • 从另一端分支切出新分支 -> 切换到新分支 -> 遴选修复节点到当前分支 -> 推送当前分支到远端；
  • 偶尔出现会单独针对某个环境从线上分支切出新分支开发，此时当修复Bug的时候，则会出现需要提交至三端分支；
  • 此时问题便暴露出来：每多一条需提交的分支，则需要重复执行遴选操作。
  • 那么需求来了：当提交好一个分支后，后续的每个分支操作是否可以有个工具帮忙执行操作，这便是本文的缘由；

效果演示

  • 文本工程使用的是：DC

  • 前期准备流程：

    1. 使用 Fork 打开当前工程，并找到 develop 分支，切出一条基底分支名为：feature/dev_cherry_pick@spirit
    2. 在这个分支上上传文件并推送到远端：
      • sh文件：\Tools\Git\git_cherry_pick.sh
      • C#代码：\Assets\DCFrame\Editor\Git\CherryPick.cs
    3. 在这个分支上，再切出一条分支：hotfix/debug@spirit
    4. 在 hotfix/debug@spirit 上传2条任意提交到远端；

  • 效果演示：

    1. 打开 Unity -> Tools -> Git -> 分支遴选工具；

    2. 填写当前分支基底和目标分支基底，都填写 develop，并点击开始执行；

    3. 等待 bash 窗口执行完毕，并打印出新分支名称；

    4. 打开 Gitlab 网页，点击合并请求 - > 新建合并请求 -> 填写源分支为新生成分支，目标分支为 develop -> 查看对比；

    5. 则发现刚才手动上传的内容，都成功遴选到新分支了；

制作流程

sh工具

  1. 从需求出发，可以得知 Git 指令是可以满足需求，其中重点所需：
    • git checkout ：切换分支
    • git checkout -b ：创建分支
    • git cherry-pick ：遴选提交
    • git push origin ：推送远端
  2. 当我们得知 Git 可以完成这些操作的时候，则需要代码去运行它，此时则需要 sh 文件的代码；
    • 代码内容内已标注好注释；
  3. 按照 效果演示 -> 前期准备流程，切换好分支；
  4. 打开 Fork 工具，点击 Console 控制台：输入指令；
  5. 输入的指令为： sh文件的路径 基底分支 目标分支 目标切出分支名
    • xxx/Tools/Git/git_cherry_pick.sh develop develop hotfix/cherry_dev_debug@spirit
    • 提示：最开始的xxx，需要工程的完整路径；
  6. 并可查看到 效果演示 bash 窗口的执行内容；

#基底分支
base_branch=$1
#目标分支
target_branch=$2
#目标切出分支
create_branch=$3
#黄色
green_color="\033[33m"
#红色
red_color="\033[31m"
#重置颜色
default_color="\033[0m"
# 获取当前分支名称
current_branch=$(git branch --show-current)
# 获取当前分支的最新提交
commits=$(git log "$current_branch" --reverse --pretty=format:"%H" "$base_branch..$current_branch")
# 计算 commits 的数量
commit_count=$(echo "$commits" | wc -l)
echo -e "\n$green_color 开始执行切分支操作 $default_color"
#切分支
git checkout $target_branch
# 从远端获取当前分支消息
git fetch origin $target_branch
# 拉取
git pull
#创建分支
git checkout -b $create_branch
echo -e "\n$green_color 创建分支成功 $default_color"
#切分支
git checkout $create_branch
addCount=1
#逐个应用这些提交到 new-branch
for commit in $commits; do
    echo -e "\n$green_color ($addCount/$commit_count)正在提交节点： $commit $default_color"
    git cherry-pick $commit
    # 检查是否有冲突发生
    if [ $? -ne 0 ]; then
        echo -e "\n$red_color 发现冲突！请查看以下状态：$default_color"
        git status
        echo -e "\n$red_color 请手动解决冲突，然后按下任意键继续... $default_color"
        read -n 1 -s
    fi
    addCount=$((addCount + 1))
done
echo -e "\n$green_color 正在推送当前分支到远程 $default_color"
git push origin $create_branch
#切分支
git checkout $current_branch
echo -e "\n$green_color ============= 切分支完成 ============= $default_color"
echo -e "$green_color 分支名：$create_branch ，成功遴选 $commit_count 条 $default_color"
# 成功退出
exit 0

Unity工具

  1. 由于需要每次都手动的输入命令行指令会有点繁琐，因此使用 Unity 的编辑器扩展工具来代替这个手动的写入；

using System;
using System.Diagnostics;
using System.IO;
using System.Text.RegularExpressions;
using UnityEditor;
using UnityEngine;
using Debug = UnityEngine.Debug;

public class GitCherryPick : EditorWindow {

	#region 定制内容

	/// <summary>
	/// sh脚本的文件路径
	/// </summary>
	public const string ShCherryPickPath = "/Tools/Git/git_cherry_pick.sh";

	/// <summary>
	/// 打开界面识别规则
	/// </summary>
	private static void RefreshWinInfo() {
		// 根据当前分支名，识别一些规则并自动帮忙填写到窗口内
	}
	
	/// <summary>
	/// 提示内容
	/// </summary>
	private void OnClickTips() {
		EditorUtility.DisplayDialog("分支遴选工具说明", "提示内容", "关闭");
	}
	
	
	/// <summary>
	/// 分析分支
	/// </summary>
	private static void AnalysisBranch() {
        // 需要根据开发实际需求来写，这里只是示例
		targetBranchName = "hotfix/cherry_dev_debug@spirit";
	}
	
	#endregion
	
    [MenuItem("Tools/Git/分支遴选工具")]
	public static void CreateCtrlWin() {
		Rect rect = new Rect(0,0,350,145);
		GitCherryPick window = (GitCherryPick)GetWindowWithRect(typeof(GitCherryPick), rect, true, "分支遴选");
		window.Show();
		GetBranchName();
	}

	/// <summary>
	/// 绘制窗口
	/// </summary>
	private void OnGUI() {
		GUILayout.Space(10);
		EditorGUILayout.LabelField("当前分支名称：" + curBranchName);
		GUILayout.Space(10);
		EditorGUILayout.LabelField("以下需手动填写");
		GUILayout.Space(6);
		curBaseBranchName = EditorGUILayout.TextField("当前分支基底：", curBaseBranchName);
		targetBaseBranchName = EditorGUILayout.TextField("目标分支基底:", targetBaseBranchName);
		GUILayout.Space(10);
		GUILayout.BeginHorizontal();
		var isTips = GUILayout.Button("提示");
		if (isTips) {
			OnClickTips();
		}
		var isClick = GUILayout.Button("开始执行");
		if (isClick) {
			GetCherryPickCommend();
		}
		GUILayout.EndHorizontal();
	}

	/// <summary>
	/// 点击创建脚本
	/// </summary>
	private void GetCherryPickCommend() {
		AnalysisBranch();
		if (!JudgeCreate()) {
			return;
		}
		string assetsPath = Application.dataPath;
		string projectRootPath = Path.GetDirectoryName(assetsPath);
		string scriptPath = projectRootPath + ShCherryPickPath;
		scriptPath = scriptPath.Replace("\\", "/");
		string strWrite = scriptPath + " " + curBaseBranchName + " " + targetBaseBranchName + " " + targetBranchName;
		var bashProcess = new Process {
			StartInfo = new ProcessStartInfo {
				FileName = "cmd.exe",
				Arguments = $"/c start bash -c \"{strWrite}; exec bash\"",
				RedirectStandardOutput = true,
				RedirectStandardError = true,
				RedirectStandardInput = true,
				UseShellExecute = false,
				CreateNoWindow = false
			}
		};
		bashProcess.Start();
		Close();
	}
	
	/// <summary>
	/// 获取分支名称
	/// </summary>
	/// <returns></returns>
	private static void GetBranchName() {
		Process gitProcess = new Process();
		gitProcess.StartInfo.FileName = "git";
		gitProcess.StartInfo.Arguments = "rev-parse --abbrev-ref HEAD";
		gitProcess.StartInfo.RedirectStandardOutput = true;
		gitProcess.StartInfo.UseShellExecute = false;
		gitProcess.StartInfo.CreateNoWindow = true;
		try {
			gitProcess.Start();
			curBranchName = gitProcess.StandardOutput.ReadToEnd().Trim();
			gitProcess.WaitForExit();
			RefreshWinInfo();
		}catch (Exception ex) {
			Debug.Log("获取分支名称时发生错误: " + ex.Message);
		}
	}

	/// <summary>
	/// 判断是否可以点击创建
	/// </summary>
	/// <returns></returns>
	private bool JudgeCreate() {
		if (curBaseBranchName.Length <= 0) {
			EditorUtility.DisplayDialog("提示弹窗", "当前分支基底暂未填写", "关闭");
			return false;
		}
		if (targetBaseBranchName.Length <= 0) {
			EditorUtility.DisplayDialog("提示弹窗", "当前目标基底暂未填写", "关闭");
			return false;
		}
		if (targetBranchName.Length <= 0) {
			EditorUtility.DisplayDialog("提示弹窗", "目标分支名称无法生成，请检查分支名", "关闭");
			return false;
		}
		return true;
	}

	/// <summary>
	/// 当前分支名称
	/// </summary>
	private static string curBranchName = "";
	/// <summary>
	/// 目标分支名称
	/// </summary>
	private static string targetBranchName = "";
	/// <summary>
	/// 当前分支基底
	/// </summary>
	private static string curBaseBranchName = "";
	/// <summary>
	/// 目标分支基底
	/// </summary>
	private static string targetBaseBranchName = "";
}

工具定制化

为了方便使用者的工具填写，因此开辟出了三个函数供开发者根据自己的工程所需进行实际的代码开发；

  1. ShCherryPickPath : sh文件的路径；
  2. RefreshWinInfo() ：在成功获取当前分支名后，会调用此函数，目的分析当前分支名，并根据一些规则，自动的帮忙填写 当前分支基底 or 目标分支基底；
  3. AnalysisBranch()：1）生成目标分支名，2）对当前分支基底 or 目标分支基底的分支名可以填写简称，由代码自动识别；
  4. OnClickTips() ：对以上定制化的内容，作出文本提示说明；

冲突遴选

  • 前期准备：效果演示中前期准备流程执行完；

  • 此时从 feature/dev_cherry_pick@spirit 再切出一条分支名为 ：hotfix/conflict@spirit

  • 在这个新分支上，提交2条与 hotfix/debug@spirit 分支相同的上传内容；但是需要文件内容改动不一致，来模拟冲突的情况；

  • 此时切换当前分支到 hotfix/debug@spirit，并打开 Unity的遴选工具界面，填写内容

  • 根据以上准备，可以猜想出从 hotfix/debug@spirit 遴选2条节点到新分支，都会产生冲突需要解决，此时点击开始执行，查看窗口显示；

    • 2个冲突，一个使用 hotfix/debug@spirit 的分支提交，一个使用 hotfix/conflict@spirit 的分支提交

  • 窗口显示冲突，并告知是 ToDo 文件冲突了；

  • 此时打开 Fork 进行冲突解决，选择 hotfix/debug@spirit 分支的提交；

  • 解决好了后，返回窗口内，按下任意键，继续执行 sh代码；

  • 此时会显示第二个冲突，并告知是 Version 文件冲突，那么回到 Fork 冲突解决，选择 hotfix/conflict@spirit 分支的提交；

  • 解决好了后，返回窗口内，按下任意键，继续执行 sh代码；

  • 此时窗口会显示切分支完成，并显示分支名；

  • 打开 Gitlab 网页，点击合并请求 - > 新建合并请求 -> 填写源分支为新生成分支，目标分支为 develop -> 查看对比；

  • 可以看到的是当前的提交记录由 hotfix/conflict@spirit 本身的2条提交 + 冲突解决选择 hotfix/debug@spirit 的一条提交；

  • 此时显示成功，代表此工具也可以支持冲突的处理；
