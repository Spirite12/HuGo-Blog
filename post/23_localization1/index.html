前言

本文主要介绍 Localization 、Lingguo 这两款插件对 Unity 多语言本地化的处理方式；

本地化模块方案

一、Localization

1.功能学习整理

  • Unity所需版本：2019.4+
  • 本次插件使用的是 1.43，2020.3+
  • 支持CSV： 导入导出
  • 菜单栏窗口的理解，Windows → Asset Management → xxx
    • Localization Scene Controls：可在编辑、运行模式中随时切换语言，或根据不同语言设置不同的资源内容；
    • Localization Tables：
      • New Table Collection : 可以查看当前工程所使用的的语言、已经新增所需的语言所属的字符串包或者资源包；
      • Editor Table Collection：
        • Asset Table：多语言资源存放的地方，支持的类型是 Object 类型的，因此继承此的都可以实现多语言内容；
          • 官方目前支持的类型有：Text、RawImage、Image、Text - TextMeshPro 、AudioSource
            • 也可支持其他的类型，下文中会具体列出如何实现
        • String Table：多语言文本存放的地方、可以实现静态和动态的文本
          • 个人想法：动态的想法需要额外创建 GroupAssets 资源文件，并开启 Smart 标识，可能会比较繁琐，
            • 可以使用 String.Format 格式实现动态赋值文本的操作， String Table中已放入 占位符格式文本便可，如：“{0}使用{1}攻击{2}”；

      private Locale cnLocale;
private Locale enLocale;
private AsyncOperationHandle asyncOperation;

void Start() {
    TestModifyLanguage();
}

private void TestModifyLanguage() {
    asyncOperation = LocalizationSettings.SelectedLocaleAsync;
    if (asyncOperation.IsDone) {
        InitializeCompleted(asyncOperation);
    } else {
        asyncOperation.Completed += InitializeCompleted;
    }
}

private void InitializeCompleted(AsyncOperationHandle obj) {
    var locales = LocalizationSettings.AvailableLocales.Locales;
    for (int i = 0; i < locales.Count; i++) {
        var locale = locales[i];
        if (locale.LocaleName == "Chinese (Simplified) (zh-Hans)") {
            cnLocale = locale;
        }
        else if (locale.LocaleName == "English (en)") {
            enLocale = locale;
        }
    }
}

void OnGUI() {
    if (GUI.Button(new Rect(0, 0, 100, 50), "中文")) {
        // 点击切换中文，此函数是切换语言函数
        LocalizationSettings.Instance.SetSelectedLocale(cnLocale);
    }

    if (GUI.Button(new Rect(0, 60, 100, 50), "英文")) {
        // 点击切换英文，此函数是切换语言函数
        LocalizationSettings.Instance.SetSelectedLocale(enLocale);
    }
}

        private readonly LocalizedStringTable stringTable = new LocalizedStringTable { TableReference = "LocalizeStringTable" };

void OnEnable() {
    stringTable.TableChanged += TableChanged;
}

void OnDisable() {
    stringTable.TableChanged -= TableChanged;
}

void TableChanged(StringTable _stringTable) {
    var strParamKey = GetLocalizedString(_stringTable, "ParamAttack");
    var strStringTest = GetLocalizedString(_stringTable, "ParamHero");
    var str = String.Format(strParamKey, strStringTest);
    Debug.LogWarning(str);
}

static string GetLocalizedString(StringTable table, string entryName) {
    var entry = table.GetEntry(entryName);
    return entry.GetLocalizedString();
}

二、Lingguo

1.功能学习整理

  • GitHub地址：点我
  • Unity所需版本：2020.3+
    • 从 package.json 文件中可知，低版本无法使用；
  • 语言包资产的理解：
    • XXPackage：当前语言管理资产（无需操作）
    • XXAssetDatabase：资源替换资产
      • 支持类型：Sprite、Texture2D、Mesh、AudioClip、Material、以及派生自 ScriptObject 的资源
    • XXStringDatabase：文本替换资产
      • 以 Key , Value , Comment 的形式显示当前语言所显示的多语言的内容；
      • ImportDictionary：默认打开的 .csv 形式的文件
        • 经尝试：也可支持 .txt 、.xlsx 格式的文件；
        • 支持导入原本有相同 Key 的时候，只做 Value、Comment 的替换处理
      • 支持：C#的String.Format 格式化处理
      • 新增：使用已有 Key 的嵌套处理，格式是：#XXX#
  • 本地化支持类型
    • 文本：XXStringDatabase 查看所引用的多语言文本；
    • 预制体：预制所需物体中挂载 Languag Game Object 脚本，并绑定相对应语言所需的预制体即可；
    • 贴图：XXAssetDatabase 添加所需多语言话的贴图资源（贴图资源需勾选 Addressable），并在Language脚本中绑定所需的物体；
      • Material 、AudioClip、Texture2D、Mesh、同样支持以上做法

2.实践记录

  • Q1：工程偶尔会报错，虽然不影响功能： Asset Packages/com.blg.gtc.lingguo/LICENSE has no meta file, but it’s in an immutable folder. The asset will be ignored
    • A：该插件的 LICENSE 文件缺少 meta 文件，从 Git 仓库的目录结构中查看，的确是没有 meta 文件的，所幸不影响功能，因此这个问题需等待作者那边上传；
  • Q2：在使用 XXStringDatabase 中的 ImportDictionaray 功能导入 csv 文件后，会出现乱码的问题
    • A：根据 这篇文章 的解释，打开 StringDatabaseEditor.cs 脚本的 OnImportDictionaray 函数，把 Encoding.UTF8 格式更改成 Encoding.GetEncoding(936) ，便可解决；
      • 原本使用的 UTF-8 是可以解析 txt 格式的文本，现在改成新格式后不支持；
      • 如果需要打开窗口不需要指定 csv 格式的，可以改 EditorUtility.OpenFilePanel 的第三个参数为 “”
    • A：根据 这篇文章 的做法，本人 Excel 的版本是2016版，没有出现截图内上方出现的保存格式，因此卸载16年版本，安装19年版本，新增 CSV UTF-8（逗号分隔）这个保存格式，用这个格式保存，便可解决乱码问题
  • Q3：本地化语言替换插件的做法是：
    • A：LanguageManager ：语言管理脚本，通过AddLanguageScript、RemoveLanguageScript 添加删除函数向管理数据内需要多语言管理的内容；
      • LingguoConfig ：配置文件，在设置语言索引值的时候，会调用 Reload 函数，该函数会先自身执行调用语言资源切换，切换成功后通知 LanguageManager 的数据管理进行重新赋值操作；
  • Q4：LanguageHelper.Translate() 函数的source 参数 在接受 #xxx# 字符串后，后续的文本无法读取出来，导致翻译错误
    • A：函数递归循环写的有问题，暂不处理；
      • 因这个问题，导致 String.Format 形式的办法也没办法解决，只能依靠插件提供的资源创建绑定 和 实例化 LanguageTemplate 脚本处理
  • Q5：在实例化多语言预制体模板的时候，会把预制件的根节点直接替换需要实例化的预制件上，导致原本根节点的tranform 参数值直接消失
    • 需要处理 LanguageGameObject 脚本，暂不处理；

3.更新想法

  • 文本
    • 应写个资源导入的函数，把各个语言的 csv / txt 文件导入到 相对应语言的 StringDatabase 文件中，减少手动操作的步骤；
    • 文本的路径处理：
      • 预制件：需要个工具把所有文本导出，提供给翻译后，导入在 stringDatabase 后，在打包的时候 以Value 匹配的形式，绑定 Language 脚本，删除手动操作的步骤；
      • 代码处理：写个 LangConst.cs 文件，以 Key-Value的形式写入文件，代码直接获取key的形式进行类 string.format 拼接实现多语言
        • 需要有个工具，将该文本的 value 导出文件提供翻译人员
  • 资源
    • 包括不仅限于：预制件、Material、AudioClip，Sprite、Texture2D、Mesh
    • Language 脚本绑定多语言的方式不便，需要手动操作。变化的是：新增工具：在导入特定文件夹的时候，以路径判断资源操作 与 添加相对应语言的 AssetDatabase 操作；

方案对比

方案 Unity版本 脚本语言 支持资源 导入导出 多语言实现原理
Localization 2019.4+ C# 插件支持： Text、RawImage、Image、Text - TextMeshPro 、AudioClip。
可扩展：继承至 Object 的都可		 XLIFF、CSV、TXT 1.获取方式：
1）在stringDatabase、assetDatabase 数据资源内，以 key-Value 的形式保存数据，获取的时候以Key的形式获取便可；
2.语言切换：
1）当调用切换语言的时候，会通知 stringDatabase(字符串数据库) 与 assetDatabase（资源数据库）进行释放当前所加载的数据；
2）切换语言数据，然后 继承至 LocalizedAssetEvent 的脚本都会收到数据更新，作出相对应的数据表现
Lingguo 2020.3+ C# 插件支持：Sprite、Texture2D、Mesh、AudioClip、Material。
可扩展：继承至 ScriptObject 的都可		 CSV、TXT 1.获取方式：
1）xxStringDatabase、xxAssetDatabase 数据资源内，以 key-Value 的形式保存数据，获取的时候以Key的形式获取便可；
2.语言切换：
1）LanguageManager 中存在需要管理多语言的数据列表；
2）在新增和移除多语言脚本时，都会往 LanguageManager 内注册/移除数据；
3） 在切换语言的时候，通知 LanguageManager 遍历数据列表，使其数据重新获取当前切换语言的数据并执行相对应操作；

方案总结

  • 这两款方案在使用上都可用：

    • Lingguo 插件优化空间比较大，距上次更新几个月前，可以作为学习代码、理解本地化操作是个不错的例子；
    • Localization 是官方推出的插件，其实从这一点的话，就给足了一定的安全感，体验过程中也有官方所内嵌的一些小工具可以配合使用，完善也挺好的；

  • 如果在新项目选择一个多语言方案的话，可以选择 Localization；

    • 资源多语言：直接走插件逻辑，不进行更改；
    • 本地表：需要对本地化表做出不同语言的读取、加载处理；
    • 新增类：
      • LocalizeMgr：项目封装好的一套方便调用的管理类；
      • LocalizeConst：以 Key - Value 形式存入当前所需要翻译的中文文本；
        • 目的：隔离从 Localization Tables 查找、插入的繁杂操作，直接从一个文本内获取，方便打开以及查找；
        • 新增工具：需要新增从当前文本导入到 stringDatabase的操作，以便后续翻译的时候可全导出 csv 文件以供翻译；
    • 预制体文本：
      • 文本替换：
        1. 在打开界面的时候获取当前语言，并对当前预制件作出相对应的文本替换；
        2. 新增 stringDatabase 用来保存预制体文本，新增工具导出所以预制体内文本，并插入到该 Database，然后由插件管理多语言；
      • 文本获取
        • Q：由于对预制体内所有文本都获取并翻译（有些文本是开发过程乱写的），到翻译人员那边需要手动先筛选一遍；
          • A：新增一套规则：在文本前添加 “@” 标识，在获取文本的时候可以剔除掉，减少人员手动审核量；
    • 工具需求：
      • 表数据导出 Excel；
      • 获取预制件、代码中的所需翻译文本并导出 Excel ；
