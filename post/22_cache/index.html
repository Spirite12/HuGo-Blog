前言

  • Unity提供了一套的本地化保存的类 PlayerPrefs，提供了 Int，Float，String 类型的支持；
    • 对于基础开发来说，这三个类型是足够支持的了；
    • 如果需要保存些复杂的数据，当然也可在 String 类型内保存 Json 的格式以达到目标；
    • 对于保存的数据都在同一份文件内；
  • 希望有个本地管理类，里面实现的功能有：
    1. 存储方式： 以系统为命名文件方式，存储数据在该文件夹内。（也可根据所需保存数据在不同的文件内，如：跟账号、跟设备、跟大区；）
    2. 开发方式：只需创建系统相对应的脚本类，写入需要保存的变量即可实现存储；
    3. 更新方式：根据某字段增加对本地存档的不读取方式，以避免出现玩家存储了有问题的本地数据；
    4. 加密方式：以防止玩家轻易打开文件找到变量，找到相对应的数据并做修改；
  • 开发目的：
    1. 查看本地存储方便；
    2. 开发只需写变量即可；
    3. 统一方便管理数据；

using System.Collections.Generic;

namespace Cache {
    public class CacheMgr {

        /// <summary>
        /// 本地记录类型
        /// </summary>
        public enum EnumSaveType {
            /// <summary>
            /// 账号
            /// </summary>
            Account = 1,
            /// <summary>
            /// 区服
            /// </summary>
            Arena = 2,
            /// <summary>
            /// 玩家ID
            /// </summary>
            PlayerId = 3,
        }

        /// <summary>
        /// 获取项目组本地记录类型值
        /// todo : 这里根据项目内实际数据进行赋值
        /// </summary>
        public static int GetSaveTypeValue(EnumSaveType enumSaveType) {
            return (int)enumSaveType;
        }

        /// <summary>
        /// 保存所有的缓存本地数据
        /// 调用时机：退游、切换账号、或定时保存
        /// </summary>
        public static void SaveAllCacheBase() {
            foreach (var cacheBase in cacheBaseList) {
                cacheBase.SaveFileData();
            }
        }

        /// <summary>
        /// 添加缓存数据
        /// </summary>
        /// <param name="cache"></param>
        public static void AddCacheBase(CacheBase cache) {
            cacheBaseList.Add(cache);
        }

        private static readonly List<CacheBase> cacheBaseList = new List<CacheBase>();
    }
}

using System;
using UnityEngine;
using Utility;

namespace Cache {
    public class CacheBase {
        /// <summary>
        /// 版本值，若文件内版本小于当前版本则不读
        /// </summary>
        public int version = 1;

        /// <summary>
        /// 本地记录类型
        /// </summary>
        public CacheMgr.EnumSaveType saveType = CacheMgr.EnumSaveType.PlayerId;

        public CacheBase() {
            className = GetType().Name;
            GetFileSave();
            CacheMgr.AddCacheBase(this);
        }

        /// <summary>
        /// 设置保存的路径
        /// </summary>
        /// <returns></returns>
        public void SaveFileData() {
            string path = GetSavePath();
            path = "" + path;
            string strSave = JsonUtility.ToJson(this);
            FileUtil.WriteFile(path, strSave);
        }

        /// <summary>
        /// 获取文件数据
        /// </summary>
        public void GetFileSave() {
            // 获取数据
            string path = GetSavePath();
            if (!FileUtil.ExistFile(path)) {
                return;
            }

            string content = FileUtil.ReadFile(path);
            // 低类当前版本则不读取
            int versionValue = FileUtil.GetIntValueByJson(content, "version");
            if (versionValue < version ) {
                return;
            }

            JsonUtility.FromJsonOverwrite(content, this);
        }

        /// <summary>
        /// 获取保存的路径
        /// </summary>
        /// <returns></returns>
        protected string GetSavePath() {
            if (savePath.Length > 0) {
                return savePath;
            }

            int number = CacheMgr.GetSaveTypeValue(saveType);
            return $"\\Record\\{className}_{number}_cache_data.txt";
        }

        /// <summary>
        /// 保存路径，不写有默认路径
        /// </summary>
        [NonSerialized]
        protected string savePath = "";

        private readonly string className;
    }
}

加密解密

  1. 目前做法：把需要存储的内容全部都转换成ASCII相对应的值，这时文件内便全是数字；
    • 扩展1：在文件内多存储一个随机数字，文件内的ASCII值都随着这个值做算法混淆；
    • 扩展2：以玩家ID为值，依次且循环获取单个数值对文件夹内的ASCII值做算法混淆；
  2. Anti-Cheat Toolkit ：一款U3D资源商店加密的插件；

问题记录

  1. Q：CacheBase 内有许多 FileUtil 文件内函数调用，并没有贴出来脚本？

    A：该脚本内容与函数名一致，只是加了些封装而已，为了减少文章长度故没有贴出；

  2. Q：CacheBase 内 FileUtil.GetIntValueByJson（自己写） 代替 JsonUtility.FromJson（API） 的原因是？

    A：在实例化 CacheBase 的时候会调用 GetFileSave 函数。FileUtil.GetIntValueByJson 该函数通过API介绍会实例化新该脚本类，会导致重复调用导致堆栈溢出。因此手动写了个解析Json某字段的函数；（没有想出更优解）

  3. Q：子类可做的事情有哪些？

    A：1）定义所需存储的变量；2）每个子类都继承 CacheBase 且都是单例；3）可重写存储路径；4）可更改父类 version 变量，达到废弃旧版本读取的操作；
