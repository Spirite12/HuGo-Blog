前言

Unity 打算让 UI Toolkit 成为新 UI 开发项目的推荐 UI 系统。不过个人感觉，鉴于目前UGUI的实际应用广泛度，会导致这项计划的推进程度并不快速。但是在Editor窗口的UI界面开发中，原先开发脚本在UI美观上有一定的限制，或者说没有像UGUI那样的方便，而UI Toolkit则可以做到这一点，因此感觉在Editor环境下UIToolkit的使用度会更占据一些优势；官方对比文档

UIBuilder

  1. 路径：Window -> UI Toolkit -> UI Builder

  2. 窗口介绍：

    1. StyleSheets：位置左上方，当前窗口是一个 uss 文件，文件的作用是存放多个样式，不同样式定义了不同的规则，当前样式会使与其匹配的元素进行统一的样式更改；
    2. Hierarchy：位置左方，当前窗口内的元素则是类似预制体内的Hierarchy窗口物体一样的结构；
    3. Library：位置左下方，组件库的意思，Standard 菜单栏下定义了官方的组件，而 Project 菜单栏下则是定义了开发者自定义的组件；
    4. Viewport：位置中间，UI可视窗口，类似预制体内的UI界面；
    5. Inspector：位置右方，当前物体的具体面板属性；

  3. 小知识：

    1. 双击 UIBuilder 菜单栏，会自动全屏当前窗口，再次双击便会还原；
    2. 如果做 Editor开发，可以在 Library 窗口右上角三个点勾选Editor Extension Authoring，则会出现 Editor 专用的组件；
    3. #开头的物体名称，代表开发者对该物体进行改动过名称；

  4. 拼凑一个界面：

    1. 创建 uxml 文件，为了使得 UIBuilder 界面能够显示初始的样式（也就是创建个空预制体），在 Project 选择一个文件夹，右击 Create -> UI Toolkit -> UI Document ，则可以创建一个初始的 uxml 文件；
    2. 双击当前新创建文件，则可以打开一个初始的界面；
    3. 当前拼凑的方式和UGUI类似，易上手，因此直接给出截图，有些物体改过名称，可以根据前方图标来识别是用了什么组件；

    using UnityEditor;
using UnityEngine.UIElements;
public class EditorWindows : EditorWindow {
    [MenuItem("Tools/UIToolkit")]
    public static void Open() {
        GetWindow<EditorWindows>("UIToolkit");
    }

    void OnEnable() {
        OnEnableInit();
    }

    void OnDisable() {
        // 向当前窗口移除图形视图
        rootVisualElement.Remove(visualElement);
    }

    private void OnEnableInit() {
        // 加载资源
        var assetVisual = AssetDatabase.LoadAssetAtPath<VisualTreeAsset>("Assets/UIDocument/Study.uxml");
        // 创建新的子节点
        visualElement = assetVisual.CloneTree();
        // 向当前窗口添加图形视图
        rootVisualElement.Add(visualElement);
    }

    private VisualElement visualElement;
}

    public class EditorWindows : EditorWindow {
    void OnEnable() {
        RenderInfo();
    }

    private void RenderInfo() {
        // 设置文本
        var txtTitle = visualElement.Q<Label>("TxtTitle");
        txtTitle.text = "Toolkit";
        txtTitle.style.unityTextAlign = TextAnchor.UpperCenter;
        // 设置 toggle 勾选
        var toggle = visualElement.Q<Toggle>();
        toggle.value = true;
        // 设置按钮点击反馈
        var btnSelf = visualElement.Q<Button>("BtnSelf");
        btnSelf.clicked += () => {
            Debug.LogWarning("你点击了按钮");
        };
        // 设置 Slider 值变更
        var slider = visualElement.Q<Slider>("Slider");
        slider.lowValue = 0;
        slider.highValue = 1;
        slider.value = 0.5f;
        slider.label = "Slider : " + slider.value;
        slider.RegisterValueChangedCallback((value) => {
            slider.label = "Slider : " + value.newValue;
        });
    }
}

CustomControls

  1. 简介：自定义组件，通俗的说类似于一些通用的预制体组件，减少开发人员在拼凑相似UI界面时的时间，可以从这个组件内直接拖拽到所需物体下便可使用；

  2. 路径：UI Builder -> Library -> Project -> Custom Controls

  3. 创建：

    1. 在 Hierarchy 面板上对想要创建成模板的物体，右击 Create Template ，此时打开一个 Win 窗口，并把当前文件命名后保存在 Resources 文件夹下（目的是为了能读取到当前资源）

    2. 创建一个 C# 脚本，代码如下，当前脚本的作用则是加载这个新创的模板数据；（创建好后，在 CustomControls 目录下则会出现当前脚本）

      using UnityEngine;
using UnityEngine.UIElements;

public class TemplateUIDocument : VisualElement {
    private readonly TemplateContainer tpContainer;
    // 自定义组件继承类
    public new class UxmlFactory : UxmlFactory<TemplateUIDocument> { }

    public TemplateUIDocument() {
        // 加载资源
        tpContainer = Resources.Load<VisualTreeAsset>("GoTemplate").Instantiate();
        // 调整全撑开
        tpContainer.style.flexShrink = 1.0f;
        hierarchy.Add(tpContainer);
    }
}

StyleSheets

  1. 简介：样式表，定义了一系列的规则，使与其匹配的元素进行统一的样式更改；

  2. 位置：UI Builder 的 左上角 Style Sheets，可新建 uss 文件或用已有 uss 文件；

  3. 小知识：

    • 鼠标放在样式上， Viewport 中会高亮已经应用当前样式的区域；

  4. 写法介绍：

    1. ElementCSharpType：
      • 可写范围： Library 下的所有控件名
      • 示例：Label ，调整参数 Text -> SIze -> 24 ，效果则是所有的Label的文本大小都变成24号；
    2. #elementNameOrId：
      • 可写范围：Hierarchy 下的所有物体 带 # 的物体名：
      • 示例：#TxtTitle，调整参数 Text -> size -> 24，效果则是名为#TxtTitle的文本大小都变成24号；
    3. .styleClassName：
      • 介绍：以“.”开头，后续跟上自定义字母，组成一个新的样式，并再该样式内定义需要更改的数据。当有元素应用了当前样式，则会采用当前样式的数据；
      • 可理解为：属性模板，引用当前模板的元素都使用当前模板类定义的数据；
      • 赠、删：在每个物体的 Style Sheet -> Style Class List 中，可以手动新增当前 Uss 中定义了一系列的以 ”.“ 开头的样式；可在已添加样式的右边点击 X，达到删除当前样式的效果；
      • 实例：新建名为 .TextSize 样式，调整参数 Text -> size -> 24，然后将此样式直接拖进一个 Label内，效果则是可看到当前 Label 文本大小变成24号；
    4. .parentClassName > .directChildClassName
      • 介绍：x > y 格式，可理解为 x 为 一级选择项， y 为二级选择项，先查找可匹配的一级内容，再查看可匹配的二级内容；
      • 实例：.TextSize > Button ，效果则是：使用了 .TextSize 样式的，Button 类型的文本大小变成24号；
    5. .styleClassName : xxx
      • 介绍：在 ：后面定义了一系列的用户行为，在用户触发这些行为的时候，则达到对应的效果
      • API文档：pseudo-classes
      • 实例：写一个名为 Label：hover样式，调整参数 Text -> size -> 24，运行Editor窗口，效果是：当鼠标移动到 Label 样式标签上的时候，则会看到当前 Label 文本大小变成24号；

Debugger

  1. 简介：Debug 窗口，方便定位、查找问题所在，方便查看元素数据；

  2. 打开路径：

    1. 菜单栏：Window -> UI Toolkit -> Debugger

    2. Editor窗口：右上角 三个点 -> UI Toolkit Debugger

  3. 快捷键：Ctrl + F5

  4. 功能介绍：

    1. 左下拉框：列出当前窗口的Item项；

    2. 右参数框：列出当前所选项的详细参数，可对参数进行更改，达到本次有效的功能，方便查找并定位问题所在；

    3. Pick Element：显示当前的对应窗口的已激活节点的结构样式（灰色外框），当选中某个节点时候，则以绿色外框高亮；

    4. Show Layout：在显示当前窗口中，鼠标移动到想查看的区域后，会自动显示当前区域的显示范围，Debuger窗口会自动选择当前节点，并列出对应参数；方便定位与查看相对应的数值；

  5. 对应文档：API

    using UnityEditor;
using UnityEngine.UIElements;

[CustomEditor(typeof(Test))]
public class TestInspector : Editor {
    public override VisualElement CreateInspectorGUI() {
        var root = new VisualElement();
        // 加载 uxml 资源并实例化到当前显示面板
        var visualTree = AssetDatabase.LoadAssetAtPath<VisualTreeAsset>("Assets/UIDocument/Study.uxml");
        visualTree.CloneTree(root);
        return root;
    }
}

  5. 问题记录：Editor窗口上有代码的设置值的行为，当前 Inspector 面板上是没有的。原因是：设置代码写在了 EditorWindow下。更好的做法应该是写一个类，然后让这两个面板直接实例化此类，这样两边就都可以进行相对应的操作；
